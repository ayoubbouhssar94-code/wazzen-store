# Backend Spec

## Framework

Use FastAPI with Python 3.12+.

Recommended packages:

- `fastapi[standard]`
- `uvicorn`
- `pydantic-settings`
- `sqlalchemy[asyncio]`
- `asyncpg`
- `alembic`
- `httpx`
- `python-dotenv`
- `orjson`
- `slowapi` or equivalent rate limiter

## Config

Use `pydantic-settings`.

Required env vars:

```text
APP_ENV=production
APP_NAME=WAZZEN API
FRONTEND_ORIGIN=https://wazzen.shop
DATABASE_URL=postgresql+asyncpg://wazzen:wazzen@wazzen_database:5432/wazzen
RUN_MIGRATIONS_ON_START=true
GOOGLE_SHEETS_WEBHOOK_URL=
META_PIXEL_ID=
META_ACCESS_TOKEN=
TIKTOK_PIXEL_CODE=
TIKTOK_ACCESS_TOKEN=
SNAP_PIXEL_ID=
SNAP_ACCESS_TOKEN=
TRACKING_ENABLED=true
```

Convert the provided internal DB URL from:

```text
postgres://wazzen:wazzen@wazzen_database:5432/wazzen?sslmode=disable
```

to async SQLAlchemy format:

```text
postgresql+asyncpg://wazzen:wazzen@wazzen_database:5432/wazzen
```

## Order Endpoint

`POST /orders`

Request:

- customer name
- phone raw/local/e164 if available
- items: product id, selected offer quantity
- upsell accepted item if any
- UTM and click IDs
- event IDs
- page URL
- user agent

Server responsibilities:

1. Validate product IDs.
2. Validate offer quantities.
3. Normalize KSA phone.
4. Recalculate all prices.
5. Create order number, e.g. `WZ-20260707-000001`.
6. Store order and line items.
7. Send Google Sheets webhook.
8. Send CAPI purchase events if tokens exist.
9. Return a clean response.

## Pricing Rules

Bundle prices:

```python
OFFER_PRICES = {
    1: 199,
    2: 279,
    3: 349,
}
UPSELL_PRICE = 99
```

Do not accept frontend-provided price as the source of truth.

## Phone Validation

Accept KSA mobile numbers only:

- Starts with `05` and 10 digits.
- Or `5` and 9 digits.
- Or `9665` and 12 digits.
- Or `+9665` and 13 chars including plus.

Normalize to:

- `local`: `05XXXXXXXX`
- `e164`: `+9665XXXXXXXX`
- `digits`: `9665XXXXXXXX`

Reject anything else with Arabic error message support.

## Google Sheets Delivery

Send a JSON payload to `GOOGLE_SHEETS_WEBHOOK_URL`.

Log every attempt in `webhook_deliveries`:

- destination
- payload hash
- response status
- response body excerpt
- success boolean
- retry count
- created/updated timestamps

If webhook fails, the order must still be stored and returned to frontend. The failure should be visible in logs/admin data.

## Tracking Event Delivery

For CAPI:

- Hash PII server-side with SHA-256.
- Never expose CAPI tokens to frontend.
- Send server events promptly after the browser event.
- Use the same `event_id` generated by frontend for browser/server dedup.
- Include IP and user agent when available.

## Startup Migrations

If `RUN_MIGRATIONS_ON_START=true`, run:

```text
alembic upgrade head
```

before starting the app process.

Use an entrypoint script or Docker command that fails fast if migrations fail.

## Health Check

`GET /health` returns:

```json
{
  "status": "ok",
  "service": "wazzen-api"
}
```

Optionally include database connectivity in a deeper `/health/db`.
